## 0.11.7
**`(fix):`** Introduces a flag `use_str_enums` to swap from using proper Enum classes to using Literals to represent enums. This change allows for forward compatibility of enums, since the user will receive the string back.

  ```
  config:
    pydantic_config:
      use_str_enums: true
  ```

  generates enums as:

  ```
  Operand = typing.Literal[">", "=", "less_than"]
  ```
