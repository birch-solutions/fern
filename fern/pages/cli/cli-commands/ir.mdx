---
title: fern ir
description: The command "fern ir" generates an Intermediate Representation (IR).
---

Use `fern ir <path-to-output>` to generate an Intermediate Representation (IR) 
of the Fern Definition. 

```bash
fern ir <path-to-output>
```

An IR is a standardized representation of an API definition that the Fern generator can read, 
with all unlisted fields in the original Fern Definition set to `null`. The IR is generally magnitudes 
larger than its corresponding Fern Definition, as seen in the example below. 

<CodeBlocks>
```yaml maxLines=16 title="Fern Definition"
service:
  auth: false
  base-path: ''
  endpoints:
    returnsAListOfUsers:
      path: /users
      method: GET
      auth: false
      display-name: Returns a list of users.
      response:
        docs: A JSON array of user names
        type: list<string>
      examples:
        - response:
            body:
              - string
```

## Options

### api

Use `--api` to specify the API to generate an IR from if you have multiple defined in your `fern/apis/` folder. 

```bash
fern ir <path-to-output> --api public-api
```

### version

Use `--version` to specify the version of IR to produce. 

```bash
fern ir <path-to-output> --version 
```

### language

Use `--language` to generate the IR for a specific language.

Choices: `typescript`, `java`, `python`, `go`, `ruby`, `csharp`

```bash
fern ir <path-to-output> --language typescript 
```

### audience

Use `--audience` to filter the IR for certain audiences. Can be multiple.

<CodeBlocks>
```bash title="one audience"
fern ir <path-to-output> --audience internal
```
```bash title="multiple"
fern ir <path-to-output> --audience internal beta
```
</CodeBlocks>
