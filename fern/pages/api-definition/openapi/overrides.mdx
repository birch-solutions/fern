title: How to use OpenAPI Overrides
description: Use overrides to customize user-facing documentation fields, examples, and naming.
---

When you generate your OpenAPI from server code, you may want to further customize portions of your OpenAPI spec. You can do this with OpenAPI Overrides. When you run `fern generate`, Fern will automatically apply these overrides on top of your OpenAPI spec.

## How to use OpenAPI Overrides

To use OpenAPI Overrides, create a file named `openapi-overrides.yml`. You can put the file anywhere in your project. This file contains the overrides you want to apply to your OpenAPI spec.

## Specifying OpenAPI Overrides in `generators.yml`

You'll need to specify the path to your `openapi-overrides.yml` file in your `generators.yml` file. Here's an example:

<CodeBlock title="../openapi-overrides.yml">
```yaml
api: 
  - path: ../openapi.json
    overrides: ../openapi-overrides.yml 
```
</CodeBlock>

## Overries for AsyncAPI

You can also use overrides for AsyncAPI when you have WebSocket APIs. To do this, create a file named `asyncapi-overrides.yml`. You can put the file anywhere in your project. This file contains the overrides you want to apply to your AsyncAPI spec.

<CodeBlock title="../asyncapi-overrides.yml">
```yaml 
channels:
  /v2/realtime/ws:
    x-fern-display-name: Streaming
```
</CodeBlock>

## Specifying AsyncAPI Overrides in `generators.yml`

You'll need to specify the path to your `asyncapi-overrides.yml` file in your `generators.yml` file. Here's an example:

<CodeBlock title="generators.yml">
```yaml
api:
  - path: ../asyncapi.yml
    overrides: ../asyncapi-overrides.yml 
```
</CodeBlock>
